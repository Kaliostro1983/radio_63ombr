# Changelog

Усі помітні зміни в проєкті фіксуються тут.

Формат записів:
- Added — новий функціонал
- Changed — зміни в існуючій поведінці
- Fixed — виправлення помилок
- Notes — технічні примітки або ручні дії

---

## [0.10.0] - 2026-03-28

### Added
- У верхній стрічці статусу (`base.html`) відображення **Git-ревізії** (короткий хеш; суфікс `+` при «брудному» дереві): читання з `.git` на кожен рендер, резерв через змінні **`APP_GIT_REVISION`** / **`APP_GIT_REVISION_FULL`** для збірок без репозиторію. Логіка в **`app/core/version.py`**, глобаль **`get_app_git_revision`** у Jinja.

### Changed
- **Еталонка** (`etalons_inner.html`, **`POST /etalons/save`**, **`POST /etalons/panel/save`**): прибрано поле «Дата початку»; дати початку/завершення задаються лише на вкладці **«Картка р/м»** (`/networks/save` → `etalons.start_date` / `end_date`). Збереження текстових полів еталонки більше не змінює `start_date`.

### Fixed
- **Режим `embed=1`** (`base.html`): приховано дубль стрічки статусу всередині iframe (вкладки «Пошук» та «Орієнтири» на `/intercepts-explorer`), залишено лише зовнішній статус-бар.

## [0.9.0] - 2026-03-21

### Added
- Документація контракту бота ↔ **`POST /api/ingest/whatsapp`**: **`docs/BOT_INGEST_API.md`** (тіло запиту, масив **`actions`**, тип **`send_message`**); посилання в **`docs/FILE_MAP.md`** та **`docs/ARCHITECTURE.md`**.
- Модальне вікно орієнтира: кнопка **«Копіювати»** біля полів **MGRS** та **WKT** (копіювання в буфер обміну).

### Changed
- **Орієнтир**: збереження без координат (порожні MGRS/WKT) узгоджено між **`landmarks.js`** та **`landmarks.py`** (`_resolve_landmark_geometry`).
- **Перегляд перехоплень** (`intercepts_explorer.js`): дозавантаження через **`IntersectionObserver`** і сентинел замість постійної перевірки на **`scroll`**; **`content-visibility: auto`** для карток; перепідключення observer при поверненні на вкладку «Перегляд»; fallback на **`scroll`** + **`requestAnimationFrame`** без `IntersectionObserver`.
- Кеш статичних ресурсів: версії **`app.css`**, **`landmarks.js`**, **`intercepts_explorer.js`**.

### Fixed
- **Пошук перехоплень** (`intercepts_search.js`): подвоєння карток через гонку **submit** і **scroll** (два паралельні запити з **`offset=0`**); єдиний прапор **`loading`** на весь **`loadData`**.

### Notes
- Поле **«Коментар»** у модалці орієнтира: збільшена висота (**`rows`** та **`min-height`**).

## [0.8.0] - 2026-03-21

### Added
- Довідник **`landmark_geoms`** (точка / зона / крива) та поля **`landmarks.id_geom`**, **`landmarks.location_mgrs`**; у формі орієнтира — вибір типу геометрії, **MGRS** для точки (конвертація в WKT у клієнті), **WKT** для зони та кривої.
- Модальне вікно орієнтира (**`base.html`** + **`landmarks.js`**): компактніша верстка, прев’ю геометрії на **Leaflet** у режимі **супутника** (Esri), підключення тайлів як у блоці пеленгів на картці р/м.
- **Перехоплення → Перегляд**: дозавантаження списку (**«Завантажити ще»** + підвантаження при скролі), у відповіді API — **`network.group_id`** для контексту р/м.
- **Перехоплення**: у результатах пошуку орієнтирів у картці — **зелений** стиль, якщо `id_group` орієнтира збігається з `group_id` радіомережі повідомлення.

### Changed
- **Граф позивних** (картка р/м і «Зв’язки»): вимкнено масштабування колесом (скрол сторінки), зменшено відштовхування вузлів у симуляції, додано **перетягування** вузлів після розкладки; зняття глобальних слухачів при перемальовуванні графа.
- **Еталонки**: прибрано заголовок на панелі, **автокомпліт** частоти/маски як на інших вкладках (`networks_autocomplete.js` підтримує два поля).

### Fixed
- **`init_db()`**: для міграції `landmark_geoms` використовується **`row_cnt[0]`** і встановлено **`row_factory = sqlite3.Row`**, щоб не падати з `TypeError` на старті застосунку.

### Notes
- Координати MGRS на сервер не конвертуються (уникнення нативної збірки `mgrs` на Windows); збереження йде з уже сформованим **POINT(WKT)** з браузера.

## [0.6.7] - 2026-03-21

### Fixed
- Логи `app.http` через stdlib могли **не з’являтися взагалі**: якщо на root немає handler, `logging.lastResort` приймає лише **WARNING+**, тож **INFO губиться**. Middleware тепер пише кожен запит напряму в **`sys.stderr`** (префікс `63ombr HTTP >>` / `<<`), без залежності від root.
- При старті додатку в stderr виводиться рядок `63ombr: OK startup — HTTP requests log as…`, щоб одразу бачити, що запущена нова версія.
- `_ensure_root_logging`: якщо на root лише `NullHandler`, очищаємо їх і додаємо `basicConfig`, щоб інші `get_logger` знову показували INFO.

- На сторінці **Перехоплення**: біля кожного позивного (хто викликає / кого викликають / згадувані) відображається іконка статусу; клік по позивному відкриває модалку редагування; після збереження статусу іконка в картці перехоплення оновлюється одразу (подія `callsignModalSaved`, пряме оновлення DOM).

### Changed
- Джерело версії: читання з `docs/VERSION` замість кореневого `VERSION`; у `create_app()` додано передачу `app_version` у `app.state.templates.env.globals`.
- API перехоплень: у відповідях з позивними використовується `callsign_status_id AS status_id`, щоб фронт отримував іконку статусу.
- Розмір фото в модалці «Особова справа»: 240×240 px.

### Fixed
- Виправлено відображення номера версії в сайдбарі (раніше показувало лише «v» через відсутність `app_version` у шаблонах, що реально використовуються).
- Оновлення іконки позивного в картці перехоплення після зміни статусу в модалці: реалізовано через глобальну подію `callsignModalSaved`, щоб не залежати від порядку реєстрації callback.

### Notes
- Модалка позивного підключена в `base.html`, тому доступна на всіх сторінках; сторінка Позивні реєструє `setCallsignModalOnSave` для оновлення таблиці, Перехоплення слухає лише подію.
