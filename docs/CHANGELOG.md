# Changelog

Усі помітні зміни в проєкті фіксуються тут.

Формат записів:
- Added — новий функціонал
- Changed — зміни в існуючій поведінці
- Fixed — виправлення помилок
- Notes — технічні примітки або ручні дії

---

## [0.3.0] - 2026-03-07

### Added
- Додано формування DOCX-звіту з даних SQLite за вибраний період.
- Додано формування DOCX-звіту зі вставленого тексту.
- Додано preview для вибірки звіту: кількість батчів і точок.
- Додано web-редагування постів.
- Додано відображення режиму формування звіту: текст або БД.

### Changed
- Інтерфейс модуля пеленгації розширено блоком формування звіту.
- Параметри періоду звіту зібрані в один ряд.
- Кнопка "Сформувати звіт" працює за правилом:
  - якщо текстове поле не порожнє — звіт формується з тексту;
  - інакше — з БД за обраний період.
- Для зв’язку `peleng_batches -> networks` використовується тільки `frequency`.
- Якщо `network` не знайдено, використовується fallback:
  `УКХ р/м НВ підрозділу, н.п. УТОЧНЮЄТЬСЯ`.

### Fixed
- Виправлено генерацію звіту з тексту: generator приведено до list перед передачею в DOCX builder.
- Виправлено помилку `latin-1 codec can't encode characters` у назві файлу відповіді HTTP.

### Notes
- Пости поки що редагуються через `posts.json`, а не через SQLite.
- `posts.json` рекомендовано додати в `.gitignore`.
- Поточна реалізація є перехідним етапом до повністю DB-native конфігурації постів.

## [0.4.0 ]

### Added
- Додано імпорт перехоплень з XLSX через web-інтерфейс.
- У модулі **Пошук перехоплень** додано кнопку **"Імпорт XLSX"** поруч із кнопкою пошуку.
- Реалізовано відкриття системного діалогу вибору файлу `.xlsx`.
- Після вибору файл відправляється на backend endpoint:
  `POST /api/import/xlsx`.

### UI
- Додано окремий JS-модуль `intercepts_import.js`.
- Реалізовано:
  - виклик hidden `<input type="file">`
  - upload файлу через `fetch` + `FormData`
  - блокування кнопки під час завантаження
  - перевірку розширення `.xlsx`

### Result Modal
- Після обробки файлу показується модальне вікно з результатами імпорту.
- Відображаються такі метрики:
  - назва файлу
  - загальна кількість рядків
  - кількість оброблених записів
  - додані записи
  - дублікати
  - пропущені записи
  - кількість помилок

## [0.4.2] - 2026-03-10

### Changed
- Парсер structured-перехоплень змінено: початок тіла повідомлення тепер визначається
  як перший непорожній рядок після поля `Відправник:`.

### Fixed
- Виправлено помилку парсера, через яку дата, `Отримувач(і)` та `Відправник`
  могли потрапляти в `body_text`.
- Парсер більше не прив’язується до шаблону `1.` на початку рядка, тому
  коректно обробляє перехоплення без нумерованих пунктів.

### Notes
- Якщо після `Відправник:` відсутній текст повідомлення, таке перехоплення
  ігнорується під час імпорту.
