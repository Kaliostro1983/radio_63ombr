# Changelog

Усі помітні зміни в проєкті фіксуються тут.

Формат записів:
- Added — новий функціонал
- Changed — зміни в існуючій поведінці
- Fixed — виправлення помилок
- Notes — технічні примітки або ручні дії

---

## [0.3.0] - 2026-03-07

### Added
- Додано формування DOCX-звіту з даних SQLite за вибраний період.
- Додано формування DOCX-звіту зі вставленого тексту.
- Додано preview для вибірки звіту: кількість батчів і точок.
- Додано web-редагування постів.
- Додано відображення режиму формування звіту: текст або БД.

### Changed
- Інтерфейс модуля пеленгації розширено блоком формування звіту.
- Параметри періоду звіту зібрані в один ряд.
- Кнопка "Сформувати звіт" працює за правилом:
  - якщо текстове поле не порожнє — звіт формується з тексту;
  - інакше — з БД за обраний період.
- Для зв’язку `peleng_batches -> networks` використовується тільки `frequency`.
- Якщо `network` не знайдено, використовується fallback:
  `УКХ р/м НВ підрозділу, н.п. УТОЧНЮЄТЬСЯ`.

### Fixed
- Виправлено генерацію звіту з тексту: generator приведено до list перед передачею в DOCX builder.
- Виправлено помилку `latin-1 codec can't encode characters` у назві файлу відповіді HTTP.

### Notes
- Пости поки що редагуються через `posts.json`, а не через SQLite.
- `posts.json` рекомендовано додати в `.gitignore`.
- Поточна реалізація є перехідним етапом до повністю DB-native конфігурації постів.

## [0.6.2] - 2026-03-20

### Added
- Додано кнопку **«Копіювати»** у картці перехоплення (копіює текст у стандартному форматі з `{caller}` та `{callees}`).
- Додано блок **швидкої ідентифікації позивного** в модальне вікно (іконки статусів; встановлює `source=РЕР`).
- Додано відображення **тегів іконками** в таблицю **«Активність р/м»** на головній сторінці.
- Додано колонку **«Теги»** (іконки) у таблицю вибору р/м на сторінці `/networks`.
- Для сторінки **«Орієнтири»**: додано чекбокс **«Активний»** та кнопку **«Видалити»** (deactivate + очищення відповідних збігів).

### Changed
- Intercepts Explorer: перехід на більш лінивий підхід завантаження деталей (щоб зменшити навантаження та затримки).
- SQLite: збільшено `timeout` / `busy_timeout` для уникнення миттєвих `database is locked`.
- Оновлено парсер/lookup для структурованих повідомлень (trim/normalized compare) для стабільнішого розпізнавання мереж.

### Fixed
- Зменшено частоту/тривалість lock-складових під час фонового matching для landmark keywords.

### Notes
- Для відображення іконок тегів використовується шлях `app/static/icons/network_tags/<tag_id>.svg` та обов’язковий fallback `_default.svg`.

## [0.5.0] - 2026-03-13

### Added
Додано підтримку пошуку за частотою або маскою у модулі Пошук перехоплень.
Реалізовано нормалізацію введеної частоти (normalize_freq_or_mask).
Поле пошуку змінено на "Частота/маска" з підтримкою форматів:
300.3010 — точна частота
300.301 — маска
300 — маска по діапазону
3003010 — автоматична нормалізація.
Додано відображення повного тексту перехоплення у результатах пошуку.
Додано lazy loading результатів при прокрутці сторінки.
Додано підсвічування знайдених фраз у тексті повідомлення.
Додано CSS стилі для сторінки пошуку перехоплень у app.css.

### Changed
Перероблено endpoint GET /api/intercepts/search.
Поле frequency тепер обробляється як частота або маска.
Результати пошуку тепер повертають text (повний текст), замість preview.
UI картки перехоплення перероблено для відображення:
дата/час
частота
опис мережі
текст повідомлення.
Інтерфейс результатів більше не обмежений шириною 600px.

### Fixed
Виправлено помилку HTTP 500 при натисканні "Пошук" без параметрів.
Виправлено помилку NameError: normalize_freq_or_mask is not defined.
Відновлено функцію normalize_nonstandard_type_1, яка використовувалась у ingest_service.
Виправлено область видимості змінної words у backend пошуку.
Усунуто конфлікт модального вікна імпорту XLSX, яке відкривалось при завантаженні сторінки.

### Notes
Нормалізація частоти винесена у app/core/normalize.py.
Сторінкові стилі інтегровані у глобальний app.css для уникнення inline CSS.
Пошук по частоті виконується через = або LIKE залежно від типу введеного значення.

## [0.6.0] - 2026-03-16

### Added
- Відображення версії додатку в сайдбарі (значення з `docs/VERSION`), посилання на сторінку зміни веде на `/changelog` (вміст `docs/CHANGELOG.md`).
- Правило в `.cursor/rules`: при фразі «зафіксуй зміни» додавати запис у `docs/CHANGELOG.md`.
- На сторінці **Еталонки**: окрема секція «Позивні (з БД)» — позивні підтягуються з таблиці `callsigns` за поточною мережею; поле «Позивні» прибрано з форми збереження.
- Спільне модальне вікно редагування позивного: винесено в `callsign_modal.js` і розмітку в `base.html`; використовується на сторінках **Позивні** та **Перехоплення**.
- На сторінці **Перехоплення**: біля кожного позивного (хто викликає / кого викликають / згадувані) відображається іконка статусу; клік по позивному відкриває модалку редагування; після збереження статусу іконка в картці перехоплення оновлюється одразу (подія `callsignModalSaved`, пряме оновлення DOM).

### Changed
- Джерело версії: читання з `docs/VERSION` замість кореневого `VERSION`; у `create_app()` додано передачу `app_version` у `app.state.templates.env.globals`.
- API перехоплень: у відповідях з позивними використовується `callsign_status_id AS status_id`, щоб фронт отримував іконку статусу.
- Розмір фото в модалці «Особова справа»: 240×240 px.

### Fixed
- Виправлено відображення номера версії в сайдбарі (раніше показувало лише «v» через відсутність `app_version` у шаблонах, що реально використовуються).
- Оновлення іконки позивного в картці перехоплення після зміни статусу в модалці: реалізовано через глобальну подію `callsignModalSaved`, щоб не залежати від порядку реєстрації callback.

### Notes
- Модалка позивного підключена в `base.html`, тому доступна на всіх сторінках; сторінка Позивні реєструє `setCallsignModalOnSave` для оновлення таблиці, Перехоплення слухає лише подію.

## [0.6.1] - 2026-03-18

### Added
- `GET /health` endpoint для readiness-перевірки.
- Скрипт `run_and_open.ps1`: запускає сервер, чекає `/health`, відкриває `/home` у браузері (щоб уникнути “немає звʼязку” при старті).
- На сторінці **Головна → Активність р/м**: візуальні покращення таблиць (центрування, “пігулки” для heatmap, показ нулів, людський формат періоду).

### Changed
- Імпорт XLSX: додано агрегацію причин пропусків (`XLSX import skip reasons breakdown`) та підсумковий лог (`XLSX import done`) на рівні `NOTICE`.
- Пошук мережі в ingest (`ensure_network`): додано `LIKE` fallback для масок/префіксів, щоб мережі краще знаходились при часткових збігах.
- Парсер template-перехоплень: частота/маска нормалізується через `normalize_freq_or_mask` перед валідацією (знімає залежність від “вже нормалізованого” формату у джерелі).

### Fixed
- Міграція тегів мереж: виправлено відновлення `network_tag_links` у випадку частково-мігрованої БД з неправильним foreign key.

### Notes
- Теги радіомереж винесено в окремий довідник `network_tags` + звʼязку `network_tag_links`; legacy звʼязка може залишатися як `network_tag_links_old` для міграції.
